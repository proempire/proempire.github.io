前期学习大纲

发表于 2017-09-15

一、前端主要语言

包括HTML，CSS及JS，简单说一下它们的功能（详细内容可参照w3school）：

HTML：超文本标记语言，浏览器渲染网页的基础元素，包括了标题，正文，图片，列表，表格等元素

CSS：层叠样式表，对HTML元素进行外观上的布局和修饰，主要包括定位，边距，文字，颜色，位移等等内容

JS：全程Javascript，浏览器通用的网页脚本语言，可对网页元素进行动态修饰。也作为项目开发架构或工具的基础语言。

二、HTML与CSS

建议第一个月把这两个东西尽可能地熟悉，资料已上传至群里，下周会有第一次统一培训。

对于HTML，可以先在一个网页文件中将所有元素全部使用一次，对于某些标签如ol与ul的区别，span与p的区别可以注意一下（涉及块级元素与行内元素的区别）

CSS方面可能比较繁琐，建议重点了解清楚盒模型margin, padding与border的含义及作用，以及float的作用。所有文本、颜色、背景内容请积极查阅手册～

说了这么多，如何做出来让自己看看呢..一个简化的HTML结构已上传到群文件，请在body部分参照w3school进行尝试～

三、JS

参考书籍已先行上传之群文件，此部分内容较多。。这边做一些简单的总结：

  • 所有JS的数据类型及它们的相互转换、自带的方法
  • 高阶函数与闭包
  • JS在动态操作HTML和CSS上的应用
  • 事件的冒泡与捕获
  • 各类人机交互事件，涵盖鼠标与键盘
  • ES6标准

使用以下工具前请事先准备node最新版本环境，这些工具都有相应的官网

vue-day2

发表于 2016-10-11

一、class与style绑定

v-bind:class接受一个对象用来动态切换class：

div class="static" v-bind:class="{ 'class-a': isA, 'class-b': isB }"></div>

data: {
  isA: true,
  isB: false
}

也可以传入数组来达到同样的效果，最新版本可在数组语法中使用对象语法。

vue可以自动添加css中一些新属性的前缀！

二、条件渲染、列表渲染

template可作为条件渲染的包装元素：

<template v-if="ok">
  <h1>Title</h1>
  <p>Paragraph 1</p>
  <p>Paragraph 2</p>
</template>

  • v-else可紧跟在v-if或v-show块后用来被识别。

  • v-if的首屏渲染时间较短，v-show反之，但后者切换快。

  • vue中包含了数组的变异方法，使其均能触发视图更新。

  • track-by在input等需要存储临时状态的组件中应慎用。

三、方法与事件处理器

v-on指令用来监听事件：

<div id="example">
  <button v-on:click="greet">Greet</button>
</div>

var vm = new Vue({
  el: '#example',
  data: {
    name: 'Vue.js'
  },
  // 在 `methods` 对象中定义方法
  methods: {
    greet: function (event) {
      // 方法内 `this` 指向 vm
      alert('Hello ' + this.name + '!')
      // `event` 是原生 DOM 事件
      alert(event.target.tagName)
    }
  }
})

也可以内联JavaScript语句来完成此流程。

事件修饰符：stop, prevent, capture, self。

key alias:

<!-- 同上 -->
<input v-on:keyup.enter="submit">

<!-- 缩写语法 -->
<input @keyup.enter="submit">

四、表单控件绑定

v-model可实现表单数据双向绑定：

<input type="checkbox" id="checkbox" v-model="checked">
<label for="checkbox">{{ checked }}</label>

五、插件

css进阶及css3培训

发表于 2016-10-10

一、css回顾

  1. 盒模型

    中心部分为content, 以外分别是padding, border和margin，三者共同构成HTML元素的盒模型

    someElement{
    display: block;
    margin: 0 auto;
    border: 1px solid red;
    padding: 20px;
}

    推荐：尽可能使用盒模型布局，少使用脱离文档流的absolute，养成良好习惯

  2. 背景

    css2中的background有五个属性，以下：

    background-color, -image, -repeat, -attachment, -position

    对后面三个作一点说明：

    background-repeat: repeat-x, repeat-y, no-repeat

    background-attachment: scroll, 默认值。背景图像会随着页面其余部分的滚动而移动。
    fixed, 当页面的其余部分滚动时，背景图像不会移动。

    background-position: 使用关键字设置的话，第一个是竖直定位，如top，第二个是水平定位。另外有百分比和尺寸设置法，

  3. 文本，字体，定位

    文本： text-align, text-decoration

    字体： 可以用一系列特定字体，最后加上通用字体：

    p {font-family: Times, TimesNR, 'New Century Schoolbook',
 Georgia, 'New York', serif;}

    （不要求）

    定位： 主要围绕position属性展开，在必要的情况下，声明此属性，值有absolute, relative, fixed等等（前两种较常用）。

    marked：两种情况下top, bottoom, left, right含义是不相同的，注意区分。

二、媒体查询

定义：媒体查询包含了一个媒体类型 和如CSS3规范中描述的包含一个或多个表达式的媒体属性，这些媒体属性会被解析成真或假。如果媒体查询中的媒体类型与文档要展示的设备相符则查询结果为真，并且媒体查询中的所有表达式为真。

<!-- link元素中的CSS媒体查询 -->
<link rel="stylesheet" media="(max-width: 800px)" href="example.css" />

<!-- 样式表中的CSS媒体查询 -->
<style>
@media (max-width: 600px) {
  .facet_sidebar {
    display: none;
  }
}
</style>
  • 逻辑操作符：and, or, only

举一个结合三者的例子：

@media only screen (min-width: 980px) and (orientation: landscape){
    //write your specific style
}

orientation: landscape表示屏幕为水平，min-width: 980px表示最小宽度为980px，and连接起来后表示此二者必须同时满足，而only screen代表只有媒体为屏幕时才会应用下面的样式，所以三者结合起来，就是一个判断条件，如果满足，即返回值为真，则应用以下的样式。

三、@font-face引入字体

很好理解，这里放一个例子大家就懂了：

<style> 
@font-face
{
font-family: myFirstFont;
src: url('Sansation_Light.ttf'),
     url('Sansation_Light.eot'); /* IE9+ */
}

div
{
font-family:myFirstFont;
}
</style>

看到src了吗？在那里引用自己的字体文件就可以了，语法格式如上。

四、css3常用属性

  • border-radius, box-sizing, background-size, background-origin

    w3c上均有相关文档，这里对box-sizing和background-size作一点补充：

  • box-sizing：默认值为content-box，即声明宽高后，padding和border将在声明的宽高范围外绘制；若声明为border-box，则前两者将在声明的宽高范围内绘制。

  • background-size：可用两个百分比设置背景大小，另有两个值cover和contain，均是等比例缩放，不同之处在于前者会扩展至占满整个边框盒，后者则是背景有边缘触碰到盒边缘即停止。

五、css3 2d、3d变换

看demo！（demo我培训时给）

六、css3过渡及动画

过渡

div
{
transition: width 2s;
-moz-transition: width 2s;    /* Firefox 4 */
-webkit-transition: width 2s;    /* Safari 和 Chrome */
-o-transition: width 2s;    /* Opera */
}

多出来的那三行是为了兼容各种浏览器，以后将会经常看到这样的写法。

对div元素绑定以下过渡：所有宽度改变的行为均不是瞬间改变，而是花费2s进行渐变，例如

div{
    width:100px;
}
div:hover{
    width: 200px;
}

div原宽为100px，悬浮后将变为200px，这个过程将用2s来渐变，可以自己试一下。

动画

@keyframes myfirst
{
from {background: red;}
to {background: yellow;}
}

  • 在@keyframes中定义一个动画，from-to表示起始和结束状态。

  • 可以用百分比具体操作每个时间点的状态：

    @keyframes myfirst
{
0%   {background: red;}
25%  {background: yellow;}
50%  {background: blue;}
100% {background: green;}
}

  • 以上这些定义动画用以下命令来应用：

    div
{
animation: myfirst 5s;
-moz-animation: myfirst 5s;    /* Firefox */
-webkit-animation: myfirst 5s;    /* Safari 和 Chrome */
-o-animation: myfirst 5s;    /* Opera */
}

    在animation属性中写上定义的动画名和动画执行时间就可以了，可以自己做一下，看看效果。

express路由

发表于 2016-10-06 
var express = require('express');
var app = express();

一、Basic Route

app.get('/', function(req, res){
    //do something...
});

也可以使用字符串模式或正则表达式区配path。

二、Route Parameters

使用:key的格式传入自定义参数，参数将被移植到req.param对象中。

三、Middleware

  1. 与koa的中间件类似，用next()调用下一个中间件

  2. 设置多个处理器的语法如下：

    var cb0 = function (req, res, next) {
  console.log('CB0');
  next();
}

var cb1 = function (req, res, next) {
  console.log('CB1');
  next();
}

app.get('/example/d', [cb0, cb1], function (req, res, next) {
  console.log('the response will be sent by the next function ...');
  next();
}, function (req, res) {
  res.send('Hello from D!');
});

四、express.Router()创建router实例

看一段代码就明白了

var express = require('express');
var router = express.Router();

// middleware that is specific to this router
router.use(function timeLog(req, res, next) {
  console.log('Time: ', Date.now());
  next();
});
// define the home page route
router.get('/', function(req, res) {
  res.send('Birds home page');
});
// define the about route
router.get('/about', function(req, res) {
  res.send('About birds');
});

module.exports = router;

就是把几个中间件集成在一个router对象里，然后导出就可以啦~~
