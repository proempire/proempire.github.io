angular4.0初体验

上一篇angular回顾不知道什么时候开的坑。。当时还是2.0版本好像，既然现在出了4.0，那就看看它现在是怎样的一种新机制吧～～

一、基础组件构成

核心组件为Component，由@angular/core引入，该组件可定义以下属性：

  • selector: 选择器，用以辨识template中的html元素
  • templateUrl: 用于链接引入的模板
  • styleUrl: 链接引入的样式
  • provider: 说明创建的服务，否则会报错

另几个库里的组件包括FormModule, RouterModule等等，前者用于input元素的双向数据绑定，像这样：

<div><label>name:</label><input [(ngModel)]="hero.name" placeholder="name"></div>

后者可在module文件里引入并调用其forRoot方法定义路由，类似这样：

RouterModule.forRoot([
    {
        path: 'heroes',
        component: HeroesComponent
    }
])

二、条件渲染与列表渲染

这两个名词第一次看到是在vue。。拿来用了一下。

条件渲染

angular提供的标识符为 ngIf, 注意 不能省略，一个例子如下：

<div *ngIf="hero">
  <h1>{{title}}</h1>
  <h2>{{hero.name}}</h2>
  <div><label>id:</label>{{hero.id}}</div>
  <div><label>name:</label><input [(ngModel)]="hero.name" placeholder="name"></div>
</div>

这边假如hero为false（或经隐式类型转换后为false）则整个div的内容将不予显示

列表渲染

标识符为 *ngFor, 以下代码对一个数组heroes中的元素迭代处理，其中处理了一个点击事件，并选择性添加selected类

<ul class="heroes">
  <li *ngFor="let hero of heroes" 
  (click)="onSelect(hero)" 
  [class.selected]="hero === selectedHero">
    <span class="badge">{{hero.id}}</span>{{hero.name}}
  </li>
</ul>

三、服务的创建方式及Promise在这里的应用

创建一个服务需要引入Injectable模块，在服务中提供一系列方法，如下：

export class HeroService {
    getHeroes(): Promise<Hero[]> {
        return Promise.resolve(HEROES);
    }
}

考虑到实际处理http请求应采用异步模式，故使用es6的promise完成异步处理（此处语法均为typescript）

Promise可以理解为类似c++中用模板实例化一个对象，此处返回改造后的promise，resolve函数定义如下：

getHeroes(): void {
    this.heroService.getHeroes().then(heroes => this.heroes = heroes);
};

括号内的箭头函数即为resolve，当请求成功后即执行此函数。

四、TypeScript语法小记

官网的angular教程中使用的为typescript语法，它是javascript的一个超集，属强类型语言，这次主要踩了一个坑～

在导出一个class时，内部的属性如使用property: type这样的格式（关键在冒号），则表示用该类型创建这样一个属性（属性值为未初始化，隐式类型转换结果为false）

但是！！如果改成=操作符就会报错，主要出现在定义一个对象类型属性时出现，观察一下编辑器的提示，会显示该属性的类性为 type of type（后面这个type为自己定义的类型），如下：

heroes = Hero;

Hero为定义好的一个类，如果这么些，heroes将会是type of Hero类型

尝试写成这样：

heroes = new Hero();

能解除报错，但需要在Hero类中写出constructor，且这样做heroes性质就不一样了
