一、web push 使用动机与原理简述

相较于移动端本地应用，web站点常常缺少一项常用的功能：推送通知。此处的推送通知一般指由浏览器实现的消息推送，换个说法，就是用户在打开浏览器时，不需要进入特定的网站，就能收到该网站推送而来的消息，例如：新评论，新动态等等。统计表明多出一个这样的消息推送将有效提高网站点击率。

那么web push究竟是怎样的一个流程呢，简单地说，可以分为三个步骤：

  1. 客户端完成请求订阅一个用户的逻辑
  2. 服务端调用遵从web push协议的接口，传送消息推送（push message）到推送服务器（该服务器由浏览器决定，开发者所能做的只有控制发送的数据）
  3. 推送服务器将该消息推送至对应的浏览器，用户收到该推送

第一步，客户端请求订阅用户，过程如下：

说明一下这三步，在第一步之前，应用服务器需要生成应用服务器密钥(application server keys)，其作用是标识该服务器，保证每次发消息推送的都是同一个服务器。然后，客户端将会请求用户授权消息推送，一旦用户授权，浏览器就会生成一个PushScription，然后这个PushScription将会被发送至服务器，存入数据库，在后面的消息推送中使用。

第二步，应用服务器发送web push协议标准的api，触发推送服务器的消息推送，其中headers必须配置正确，且传送的数据必须是比特流。

应用服务器发送消息推送请求（目的是为了将更新推送到用户的浏览器），为了向推送服务器发出请求，需要查看先前获得的PushScription，取出其中的endpoint，即为推送服务器配置给该用户的访问点。

一个PushScription对象如下：

{
  "endpoint": "https://random-push-service.com/some-kind-of-unique-id-1234/v2/",
  "keys": {
    "p256dh" :
"BNcRdreALRFXTkOOUHK1EtK2wtaz5Ry4YfYCA_0QTpQtUbVlUls0VJXg7A8u-Ts1XbjhazAkj7I99e8QcYP7DkM=",
    "auth"   : "tBHItJI5svbpez7KI4CCXg=="
  }
}

其中的endpoint包含了推送服务器域名，path后面的部分为推送服务器为每个用户分配的一个标识符。

发送数据时，数据必须编码（出于安全性考虑）。推送服务器在接收到这样一个请求之后，立即开始监听用户浏览器是否处于在线状态，若是，则将消息推送发送至浏览器。

第三步，浏览器端接收消息推送，触发push事件并展示

浏览器在接收到推送服务器发来的推送后，将其解码并触发一个push事件。Service Worker由于它可以在浏览器页面未打开，浏览器未打开时执行，因此一般选择它完成web push的最后一步，即响应push事件完成展示通知等业务逻辑。

二、web push实现细节

按照上一部分所说，首先进行用户订阅。

首先注册一个Service Worker，若注册成功，返回的Promise为resolve状态，如下：

function registerServiceWorker() {
  return navigator.serviceWorker.register('service-worker.js')
  .then(function(registration) {
    console.log('Service worker successfully registered.');
    return registration;
  })
  .catch(function(err) {
    console.error('Unable to register service worker.', err);
  });
}

随后测试window环境下是否有Notification对象（此处以chrome为例，若使用firefox，uc等浏览器，需要遵循其相应标准，调用对应对象方法或引入JS SDK包），测试成功，调用Notification.requestPermission请求用户授权发送推送，若授权成功，将会返回’granted’。

接下来要做的就是使用注册好的Service Worker对象，调用pushManager.subscribe方法，从客户端获得刚刚所说的PushScription对象。

function subscribeUserToPush() {
  return navigator.serviceWorker.register('service-worker.js')
  .then(function(registration) {
    const subscribeOptions = {
      userVisibleOnly: true,
      applicationServerKey: urlBase64ToUint8Array(
        'BEl62iUYgUivxIkv69yViEuiBIa-Ib9-SkvMeAtA3LFgDzkrxZJjSgSnfckjBJuBkr3qBUYIHBQFLXYp5Nksh8U'
      )
    };

    return registration.pushManager.subscribe(subscribeOptions);
  })
  .then(function(pushSubscription) {
    console.log('Received PushSubscription: ', JSON.stringify(pushSubscription));
    return pushSubscription;
  });
}

userVisibleOnly是为了保证推送对用户可见，application server key则如前文所说，是推送服务器用以识别应用服务器的密钥，这里的密钥包含了公钥和私钥，传输的是公钥。同时，PushScription的endpoint也是在这个过程中生成的，生成公钥和私钥可以使用web-push库。

这里再次说明一下推送服务器的不可选择性，在调用subscribe生成PushScription时，浏览器会向它指定的中转服务器发送请求来生成endpoint和其余部分，这是没法控制的。

PushScription中的auth和p256dh是用来控制带载荷的push message的。

获取到PushScription对象后，将其发往应用服务器，此处简化了存储，使用nedb存下PushScription并返回Promise：

function saveSubscriptionToDatabase(subscription) {
  return new Promise(function(resolve, reject) {
    db.insert(subscription, function(err, newDoc) {
      if (err) {
        reject(err);
        return;
      }

      resolve(newDoc._id);
    });
  });
};

存储完毕后，接下来就是开发后台管理逻辑，使得管理员能够触发向用户推送消息的事件，应用服务器所做的逻辑就是遍历在数据库中存储的所有PushScription并推送消息，以下是使用web-push库完成配置密钥及联系邮箱的示例：

gulp初探

发表于 2018-01-19

考虑了很久，到底要写总结式的博客呢，还是写探索式的博客呢..然而这次又放纵了一把，来看看gulp吧～

一、gulp做什么

如果你用过webpack的话，应该对文件打包、转义、压缩等过程有了一定的了解，相比于webpack，gulp同样也是对项目进行整合处理的一个工具，但它的侧重点和前面提到的那几个webpack功能不同，它做的主要是“自动化”，也就是把你的工作流整合起来，但是在这个过程中你同样可以使用各种插件，比如压缩啊什么的。

二、指定流的源和输出

gulp.src

gulp.src接收一个url（事实上接收的是一个glob模式），从而输出一个vinyl文件流，用于之后的pipe。

node-glob与vinyl可查阅相关文档：node-glob vinyl

vinyl其实是文件的一种描述方式，与我们之前关注的path和content两个属性其实是相通的。

gulp.dest

在vinyl的官方文档中有这样的描述：src指令产生vinyl对象而dest指令消耗vinyl对象，这其实就是一个输入-输出的过程。

在指定输出目录时，一般采用相对路径，举个栗子：

gulp.src('./client/templates/*.pug')
  .pipe(pug())
  .pipe(gulp.dest('./build/templates'))
  .pipe(minify())
  .pipe(gulp.dest('./build/minified_templates'));

此处的输入指定了base，也就是相对地址的基址为./client/templates，那么dest指定的地址里就会将src中基址之后的部分拼接在dest的相对地址之后，得到的会是./build/templates/*.pug

三、创建工作流

核心：gulp.task，继承自undertaker，目的在于创建能串行(series)执行也能并行(parallel)执行的任务，gulp.task要求传入的执行函数必须遵循async-done，以标志异步函数何时执行完毕，下面看看这些异步支持是如何做的：

接收回调

var del = require('del');

gulp.task('clean', function(done) {
  del(['.build/'], done);
});

// use an async result in a pipe
gulp.task('somename', function(cb) {
  getFilesAsync(function(err, res) {
    if (err) return cb(err);
    var stream = gulp.src(res)
      .pipe(minify())
      .pipe(gulp.dest('build'))
      .on('end', cb);
  });
});

回调cb由是否接受到Error对象来判断任务是否成功～

返回一个Promise

var Promise = require('promise');
var del = require('del');

gulp.task('clean', function() {
  return new Promise(function (resolve, reject) {
    del(['.build/'], function(err) {
      if (err) {
        reject(err);
      } else {
        resolve();
      }
    });
  });
});

要实现串行和并行执行，就使用gulp的parallel和series方法，它们均接收任务（集合）作为参数，区别在于执行的方式不同。

四、任务监听

gulp.watch接收三个参数：glob模式（数组），配置参数及回调，如：

gulp.watch('js/**/*.js', gulp.parallel('concat', 'uglify'));

参数中有几个点值得说一说：

opts中的queue属性：在处理长任务时，此选项的值应为true，作用是当某个函数（任务）在执行时，将此函数推入队列中。

fn回调：当监听器发布改变（change），添加（add）或解链（unlink）事件时，在200毫秒的防反跳事件后，就会执行缓存在队列中的子调用（subsequent calls）。这里的回调需要遵循一个上面提到的规则：回调函数必须传入一个callback参数，在回调执行完毕后调用，当然，也可以采用aysnc-done的其他策略，这在上文已经提及。

watch方法返回一个chokidar（chokidar是一个对node fs模块的监听与事件进行包装的库），通过直接对这个chokidar添加事件监听，可以进行更多的控制：

var watcher = gulp.watch('js/**/*.js', gulp.parallel('concat', 'uglify'));
watcher.on('change', function(path, stats) {
  console.log('File ' + path + ' was changed');
});

watcher.on('unlink', function(path) {
  console.log('File ' + path + ' was removed');
});

node.js api文档小记

发表于 2017-12-05

一、EventEmitter

构造与添加监听器

该对象由node的event模块提供，扩展它可以获得一个EventEmitter类，实例化后即可添加监听器

const EventEmitter = require('events');

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();
myEmitter.on('event', () => {
  console.log('an event occurred!');
});
myEmitter.emit('event');

触发事件在最后一行，使用emit方法即可。

在事件处理器中（若是常规事件被调用），this默认指向事件绑定的EventEmitter，但如果使用了箭头函数，因为它遵循词法作用域，this将会指向一个空对象。

异步与同步

EventEmitter在默认情况下会同步执行所有的监听器，若想采用异步模式，使用setImmediate()或process.nextTick()方法

其他支持的功能

  • 监听器可以被调用多次，也可以只被调用一次，例如once()接口
  • EventEmitter绑定和销毁监听器时会对应地触发newListener和removeListener事件
  • 此外还有对监听器数量、名称、上限等作操作的各种接口，详情参见node.js官方文档

二、Stream

分类

  • Stream包括Readable Stream, Writable Stream, Duplex Stream和Transform Stream，前两种顾名思义，第三种指代既可读又可写的流，第四种则是可以修改或转换的Duplex Stream
  • 对象模式：流在默认情况下以Buffer, String或Uint8Array形式传输，但可以通过objectMode参数修改为对象模式，对象模式接受除null以外的其他数据类型

缓存及其意义

缓存容量由highWaterMark标识，避免内存压力过大，同时读和写有各自的缓存区，保证在速度有所差别的情况下，它们能各自独立工作。

可写流

  • HTTP requests, on the client
  • HTTP responses, on the server
  • fs write streams
  • zlib streams
  • crypto streams
  • TCP sockets
  • child process stdin
  • process.stdout, process.stderr

Writable类实现了多个控制读操作的方法，举drain方法为例，它在重新允许将数据写入流时触发（Writable.write()返回值即将由false变为true）：

function writeOneMillionTimes(writer, data, encoding, callback) {
  let i = 1000000;
  write();
  function write() {
    let ok = true;
    do {
      i--;
      if (i === 0) {
        // last time!
        writer.write(data, encoding, callback);
      } else {
        // see if we should continue, or wait
        // don't pass the callback, because we're not done yet.
        ok = writer.write(data, encoding);
      }
    } while (i > 0 && ok);
    if (i > 0) {
      // had to stop early!
      // write some more once it drains
      writer.once('drain', write);
    }
  }
}

这段官方demo大约是做了这么一件事：将data写出一百万次，若是正常结束就执行callback回调，若是被迫中止（back pressure），则触发drain事件～

此外还有cork将写入的数据由缓存全部存入内存，直到调用Writable.uncork()时被重新激活

可读流

其包含的流类型与可写流类似，需要讨论的是它的两种模式，三个状态～

两种模式：包含flowing和paused，flow模式下，通过EventEmitter发布的data事件，数据将自动进行读写，速度快；paused模式下，需要显式地调用stream.read()方法来指定读取的数据块。

三个状态：指的是readable._readableState.flowing的状态，初始为null，没有消费者，因此数据也不会生成，但当模式变为flowing后它的值也会切换为true，启动生成数据-发布data事件的流程。

关于vue、vuex的一些初步小结

发表于 2017-12-04

一、Vue组件的创建

一般语法：

Vue.component(tagName, options)
  • 务必在根组件实例化之前注册组件

组件options说明：

  • data: 必须是一个函数，目的在于返回独立的对象，保证组件之间的数据不互相影响
  • components: 局部注册一个组件，仅在当前组件作用域内可用
  • props: 用于父组件向子组件传递数据时使用，可传静态属性，也可传动态的（数据绑定，父组件数据变化引起子组件数据相应变化），一次性传递所有属性可采用v-bind指令
  • computed: 计算属性，在部分情况下可以代替watch的功能，也可用于:class的动态绑定
  • watch: 即属性侦听器，接受参数newData, oldData，含义顾名思义
  • methods: 定义组件内的方法
  • 生命周期函数：beforeCreate, created, beforeMount, mounted, updated, beforeDestroy, destroyed，其具体含义可参照官方文档

二、组件间数据传递

来看看文档里一段值得参考的语法（位置：组件-自定义事件-.sync修饰符）：

<comp :foo="bar" @update:foo="val => bar = val"></comp>
  • :foo=”bar”即父组件向子组件传递属性foo，vue中采用单向数据流，这里是为了让父组件的数据和数据的改动（若有的话）传递到子组件上，供子组件使用
  • update为vue生命周期函数，在数据变化前调用
  • 这里update部分事实上为v-on的缩写，作用是监听子组件中foo的改变并调用对应的处理器
  • 子组件向父组件发布事件(emit)会触发父组件设置的监听器（如果你设置了的话）

好了，这里涉及的概念可能比较多，所以再多解释一下吧～为了逻辑上更加清新，vue是不推荐子组件改变父组件的数据的，主要是出于维护成本的考虑，这就是所谓的单向数据流。父组件可以通过改变prop影响子组件数据，而子组件可以通过emit触发父组件方法。

但是，如果这样的数据传递不仅仅限于父子组件之间，而是祖先与后代组件或是兄弟组件呢？显然这会是一件很麻烦的事，为了处理这种多状态、多通信的情形，我们引入了vuex。

三、vuex的使用

如果你之前使用过flux、redux等状态状态管理工具的话，那么应该会很容易理解这个工具是要做什么～

下面依次介绍一下state、mutation和action这三个核心概念：

state

是一个持久化存储的状态，不归属于vue的某个特定组件，除非刷新页面，否则数据不会消失～

在根实例中注册store，下发到所有子组件，使用如下：

computed: {
  count () {
    return this.$store.state.count
  }
}

如果想要简化定义，使用mapState辅助函数就可，使用时从vuex引入该方法就可。

mutation

state的值不能直接被操作，想要改变它必须通过提交mutation的形式进行，方式为：

store.commit('something', payload)

注意一下第二个参数，这是以载荷形式提交参数，参数最多一个，想传多个参数的话必须以对象的形式提交。

mutation只能是同步函数，原因是devtool无法捕捉异步函数的快照。

action

为了执行异步操作，vuex引入了action来完成这一工作，调用方式是用dispatch方法，简化写法与state和mutation是类似的，值得看看的是将它和es6的promise和es7的async与await结合起来的工作原理：

actions: {
  actionA ({ commit }) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        commit('someMutation')
        resolve()
      }, 1000)
    })
  }
}

这个action返回了一个Promise对象，可以在后续进行处理，同时action中也可以调用其他异步action。

除了这三个概念外，还有getter（类比计算属性，用于从state派生出一些值）、module（分割较大的状态树，便于管理）。

处理表单可手动监听input、change或是使用带setter的双向绑定计算属性。

koa学习小记

发表于 2017-11-09

一、核心概念

  • 中间件：采用es6中generator方式来实现一种级联的逻辑，在一系列app.use(function* (next){…})中，一旦在某处遇到了yield next执行语句，那么将会把控制权交给下一个函数，直至执行完毕后返回为止
  • context上下文：主要封装了node的request和response两个对象，在每个request请求中被创建，中间件中用this引用
  • 几个可能用到的属性，req.query，res.status

二、koa-router

觉得形式上和express路由类似，其中router.all()区配所有http方法

支持嵌套路由，形式如下：

var forums = new Router();
var posts = new Router();

posts.get('/', function (ctx, next) {...});
posts.get('/:pid', function (ctx, next) {...});
forums.use('/forums/:fid/posts', posts.routes(), posts.allowedMethods());

// responds to "/forums/123/posts" and "/forums/123/posts/123" 
app.use(forums.routes());

其中forums为外层路由，forums.use()第二个参数传入了内层路由posts，使得路由区配的为两个url的拼接

也可以指定公共前缀如下：

var router = new Router({
  prefix: '/users'
});

router.route()将返回一个分派路由的中间件

首先是git clone，加入一个新项目时首先执行的会是这个命令，那么它做了什么呢？简单地说，就是拷贝一份远程仓库的代码到本地。

然后是对于远程主机与本地之间的一些关系处理：

如果在本地创建了项目，想部署到github上，可以考虑使用添加主机命令git remote add <主机名>，可以管理多个远程主机。

接下来有两件事：

一是同步远程的更新，常用的指令有git pull和git fetch，那么它们有什么差别呢，简单地说，git pull就是git fetch与git merge的整合；

  • 千万注意git merge后的分支是要被合并过来的分支，千万不要搞反了

第二件事是确认追踪关系，手动设置指令为–set-upstream，git clone命令拷贝下来的分支自动与远程主机建立追踪关系。

二、时间线操作

一些有用的操作：git status查看工作区文件状态，git diff比对修改内容，git log记录操作

  • 一个小细节：如果git reset回退版本并且加上了–hard参数，还能回退回来吗？答案是肯定的，版本的切换可以直接用修改指针指向来进行，所以关键就在于找到某版本对用的commit id，若找不到，可以用git reflog寻找
  • 两个小细节：撤销暂存区修改使用git reset HEAD ，撤销工作区修改使用git checkout –

vuex小记

发表于 2017-10-29

一、状态管理工具的使用动机

类似于flux、redux，vuex也是一个关于全局里所有组件的状态管理工具，目的是使得通信和状态改变更加优雅和可维护，vuex的状态同样是响应式的。

是否使用应该考虑该SPA的规模，一般考虑在中大型页面使用。

参考一下vuex官方文档的示意图：

二、Store

定义上级vue元素时可显式指定Store对象使其在子组件中可用，子组件访问方式使用this.$store

computed: {
    count: function() {
        return this.$store.state.count
    }
}

关于mapState的补充说明：

事实上它做的是一个方法的映射，将store文件中的属性映射到一个子组件里的别名上，目的是为了避免诸如this.$store.state…这样冗长的写法。

三、Getter和Mutation

在store的getter方法中完成定义，将暴露为一个属性。

// ...
getters: {
    getTodoById: (state, getters) => (id) => {
        return state.todos.find(todo => todo.id === id)
    }
}

若要传参，可返回一个函数

设置Mutation与绑定事件写法是类似的，但是触发Mutation必须用commit方法。Mutation与getter类似，第一个参数传入state：

mutations: {
    increment (state, n) {
        state.count += n
      }
}
  • Mutation的改变必须是同步函数，因为异步函数的状态无法追踪，也无法得到快照
  • 组件中提交mutation同样调用父元素的$store，当然也可以使用mapMutation来做

四、Action和Module

  • 相比于Mutation，action支持异步操作，分发指令与redux类似，使用store.dispatch()方法即可
  • vuex支持命名空间模块，命名空间要访问global assets可以在提交mutation或分发action时带上第三个参数{root: true}

  • 注册模块可用来绑定其他引入的插件，例如vue-router:

    // 注册模块 `vue-router`
store.registerModule('vue-router', {
  // ...
})

  • 模块重用应使用函数来声明状态，此处与vue的data返回方式是一致的

es6回顾

发表于 2017-10-27

（此博客内容主要参考自阮一峰老师的es6书籍）

一、环境配置和变量声明

一般采用babel工具将es6代码转为es5代码，以保证较好的兼容性，babel配置有两点值得注意一下：

  1. babel提供不同阶段语法提案的转码规则
  2. babel.cli提供node编译环境

es6的let变量赋值仅在当前块内有效

在与闭包相关的循环变量无效问题下，let操作符可解决问题（基于以上特性）

  • es6的for循环声明部分和循环体部分是父子作用域关系
  • let不允许变量提升
  • 函数表达式在es6标准下声明会被提升
  • const与let特性基本一致，但不变的是变量地址，故声明常量对象可能会有问题

以下是几个例子：

var a = [1, 2, 3, 4, 5, 6];
for (let i = 0; i < 6; i++) {
    a[i] = function() {
        console.log(i);
    }
}
a[2]();

这里的a[2]能正常输出2，但es5的var就不行了。

另外需要注意一下let和const定义引起的死区问题，保证变量声明后使用。

二、变量解构赋值

可类比为模式区配（注意一下对象的解构赋值），几个细节的演示如下：

let [a, [b, c]] = [1, [5, 3]];
console.log(a, b, c);

可嵌套赋值

解构对象可以指定默认值，但必须保证未指定的值严格等于undefined才可。

let {foo, bar: cat} = {foo: 'xyz', bar: 'kkk'};
console.log(foo, cat);

let {cont: {wrap: {line}}} = {
    cont: {
        wrap: {
            line: 'find me?',
            payload: 'padding'
        }
    }
}
console.log(line);

对象的解构赋值机制如上，后半部分为多重嵌套。

在es6中常常有这样的写法：

let {prop1, prop2, prop3} = someObj;

其内在机制即为对象的解构赋值，凡是实现了iterator接口的类型都可以进行解构赋值。

解构赋值实践：

例如函数返回多值；利用指定默认值方法完成短路运算符的一些功能

三、字符串和正则等的扩展

  • es6的String类提供了几个新功能，包括对双字节的支持、编解码、查询子串等功能
  • 为较好地兼容其他语言，可以适当利用标签模板(tag module)的raw属性
  • es6的正则添加了u和y修饰符，分别作编码扩展和粘连匹配使用
  • 组区配使用()，利用具组名区配方便地获得区配结果

函数扩展：

  • 箭头函数的实现机制事实上是提前绑定this对象，不允许es5改变上下文的几个方法
  • es6标准要求实现尾调用/递归优化，以提高性能

  • 柯里化也可实现尾递归，如下：

    let dproto = function(cb, n) {
    return function(m) {
        return cb.apply(this, [m, n]);
    }
};

let dtest = function(m, result) {
    if (m === 0) { return result; }
    return dtest(m-1, result+m);
};

let illu = dproto(dtest, 0);

    一个累加器的柯里化

四、proxy代理

配置handler可拦截默认行为，例如拦截一个默认函数调用：

let target = function(dprops) {
    console.log(`This is the ${dprops} message!`);
}

let handler = {
    apply: function(target, ctx, args) {
        Reflect.apply(target, ctx, ['no-' + args[0]]);
    }
}

let proxy = new Proxy(target, handler);

proxy('real');

修改事件参数，这样就输出了修改后的结果

proxy还有get, set, defineProperty等方法，在Reflect中均有对应。

五、Promise和Generator小记

  • Promise对象的回调函数默认在本轮事件循环全部结束后，下一轮才执行，但Promise.resolve()返回的Promise在本轮结束时进行

  • Promise.all执行时优先使用子Promise的处理函数进行处理，如下例：

    const p1 = new Promise((resolve, reject) => {
    resolve('hello');
}).then(result => {console.log(result)})

const p2 = new Promise((resolve, reject) => {
    throw new Error('p2\'s error!');
})
.then(() => console.log('p2 successful!'))
// .catch(e => console.error(e));

Promise.all([p1,p2])
.then(result => {console.log(result)})
.catch(e => console.error(e))

  • 若想将同步与异步事件分离，可考虑async方法，或用模拟try的Promise.try包装代码

  • 类数组部署遍历器可直接引用数组遍历器，如下：

    NodeList.prototype[Symbol.iterator] = Array.prototype[Symbol.iterator];

  • 扩展运算符，解构赋值均调用iterator接口

  • 数组、Map和Set均提供了三个迭代器：entries(), keys(), values()
  • next方法可带一个参数作为上一个yield的返回值，故第一个next()不必带参数

遍历Generator产生的迭代器时，for..of..不会遍历到return后的值，此时需要用yield*来获取。

补充：

  • 回调函数将err放在第一个的理由：异步函数执行分两阶段，第一阶段抛出错误的执行上下文在后面会丢失

angular4.0初体验

发表于 2017-10-05

上一篇angular回顾不知道什么时候开的坑。。当时还是2.0版本好像，既然现在出了4.0，那就看看它现在是怎样的一种新机制吧～～

一、基础组件构成

核心组件为Component，由@angular/core引入，该组件可定义以下属性：

  • selector: 选择器，用以辨识template中的html元素
  • templateUrl: 用于链接引入的模板
  • styleUrl: 链接引入的样式
  • provider: 说明创建的服务，否则会报错

另几个库里的组件包括FormModule, RouterModule等等，前者用于input元素的双向数据绑定，像这样：

<div><label>name:</label><input [(ngModel)]="hero.name" placeholder="name"></div>

后者可在module文件里引入并调用其forRoot方法定义路由，类似这样：

RouterModule.forRoot([
    {
        path: 'heroes',
        component: HeroesComponent
    }
])

二、条件渲染与列表渲染

这两个名词第一次看到是在vue。。拿来用了一下。

条件渲染

angular提供的标识符为 ngIf, 注意 不能省略，一个例子如下：

<div *ngIf="hero">
  <h1>{{title}}</h1>
  <h2>{{hero.name}}</h2>
  <div><label>id:</label>{{hero.id}}</div>
  <div><label>name:</label><input [(ngModel)]="hero.name" placeholder="name"></div>
</div>

这边假如hero为false（或经隐式类型转换后为false）则整个div的内容将不予显示

列表渲染

标识符为 *ngFor, 以下代码对一个数组heroes中的元素迭代处理，其中处理了一个点击事件，并选择性添加selected类

<ul class="heroes">
  <li *ngFor="let hero of heroes" 
  (click)="onSelect(hero)" 
  [class.selected]="hero === selectedHero">
    <span class="badge">{{hero.id}}</span>{{hero.name}}
  </li>
</ul>

三、服务的创建方式及Promise在这里的应用

创建一个服务需要引入Injectable模块，在服务中提供一系列方法，如下：

export class HeroService {
    getHeroes(): Promise<Hero[]> {
        return Promise.resolve(HEROES);
    }
}

考虑到实际处理http请求应采用异步模式，故使用es6的promise完成异步处理（此处语法均为typescript）

Promise可以理解为类似c++中用模板实例化一个对象，此处返回改造后的promise，resolve函数定义如下：

getHeroes(): void {
    this.heroService.getHeroes().then(heroes => this.heroes = heroes);
};

括号内的箭头函数即为resolve，当请求成功后即执行此函数。

四、TypeScript语法小记

官网的angular教程中使用的为typescript语法，它是javascript的一个超集，属强类型语言，这次主要踩了一个坑～

在导出一个class时，内部的属性如使用property: type这样的格式（关键在冒号），则表示用该类型创建这样一个属性（属性值为未初始化，隐式类型转换结果为false）

但是！！如果改成=操作符就会报错，主要出现在定义一个对象类型属性时出现，观察一下编辑器的提示，会显示该属性的类性为 type of type（后面这个type为自己定义的类型），如下：

heroes = Hero;

Hero为定义好的一个类，如果这么些，heroes将会是type of Hero类型

尝试写成这样：

heroes = new Hero();

能解除报错，但需要在Hero类中写出constructor，且这样做heroes性质就不一样了
